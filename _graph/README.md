# Z-ai-platform Graph Data API

> **Audience:** External tools (especially `Z-ai-graph-viewer`) that need to
> display the true state of the Z-ai-platform ID graph.
> **Updated:** On every push to `main` (automated via CI).
> **Format:** JSON (deterministic — same input produces same output).

---

## Endpoints

### 1. Full ID graph (for graph visualization)

```
GET https://raw.githubusercontent.com/stsgs1980/Z-ai-platform/main/docs/_graph/id-graph.json
```

**Content-Type:** `application/json`
**CORS:** Supported (raw.githubusercontent.com sends permissive CORS headers)
**Cache-Control:** GitHub serves with long cache lifetime; pin with `?ts=<commit-sha>` if needed

## How to consume

### JavaScript / TypeScript (e.g. Z-ai-graph-viewer)

```typescript
const URL =
  "https://raw.githubusercontent.com/stsgs1980/Z-ai-platform/main/docs/_graph/id-graph.json";

interface GraphData {
  version: string;
  effective_date: string;
  exported_at: string;
  summary: {
    ids_extracted: number;
    related_edges: number;
    aligned_with_edges: number;
    hard_pass: number;
    hard_fail: number;
    warnings: number;
  };
  ids: Array<{
    id: string;
    prefix: string;
    domain: string;
    number: string;
    file: string;
  }>;
  related_edges: Array<{ from: string; to: string; file: string }>;
  aligned_with_edges: Array<{ left: string; right: string; file: string }>;
}

export async function fetchGraph(): Promise<GraphData> {
  const response = await fetch(URL);
  if (!response.ok) throw new Error(`HTTP ${response.status}`);
  return response.json();
}
```

### React (Z-ai-graph-viewer example)

```tsx
import { useEffect, useState } from "react";
import { fetchGraph } from "./api";

export function GraphViewer() {
  const [graph, setGraph] = useState<GraphData | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    fetchGraph()
      .then(setGraph)
      .catch((e) => setError(e.message));
  }, []);

  if (error) return <div>Error: {error}</div>;
  if (!graph) return <div>Loading...</div>;

  return (
    <div>
      <h1>Z-ai-platform ID Graph</h1>
      <p>
        {graph.summary.ids_extracted} IDs, {graph.summary.related_edges} edges
      </p>
      <p>Last updated: {graph.exported_at}</p>
      <GraphSVG nodes={graph.ids} edges={graph.related_edges} />
    </div>
  );
}
```

## Refresh strategy

| Frequency          | Method                                   | When to use                |
| ------------------ | ---------------------------------------- | -------------------------- |
| On every page load | Direct fetch + browser cache             | Live dashboard             |
| **Every 5 min**    | `setInterval` + countdown UI             | **Recommended default**    |
| Every 1 hour       | fetch with `Cache-Control: max-age=3600` | Slightly delayed dashboard |
| On demand          | Manual refresh button                    | Admin tools                |
| Never (static)     | Pre-fetched JSON in build step           | SSG sites                  |

**For Z-ai-graph-viewer:** periodic refresh every 5 minutes is recommended.
It balances freshness (CI takes ~3 min from push to JSON ready) with server load
(GitHub raw.githubusercontent.com has rate limits).

### Recommended pattern: periodic refresh with countdown

```tsx
// Z-ai-graph-viewer/src/hooks/useGraphWithAutoRefresh.ts
import { useEffect, useState, useCallback, useRef } from "react";

const REFRESH_INTERVAL_MS = 5 * 60 * 1000; // 5 minutes
const STALE_THRESHOLD_MS = 15 * 60 * 1000; // mark as "stale" after 15 min

interface GraphState {
  data: GraphData | null;
  status: "idle" | "loading" | "success" | "error" | "stale";
  lastUpdated: Date | null;
  lastError: string | null;
  nextRefreshIn: number;
}

export function useGraphWithAutoRefresh(url: string) {
  const [state, setState] = useState<GraphState>({
    data: null,
    status: "idle",
    lastUpdated: null,
    lastError: null,
    nextRefreshIn: REFRESH_INTERVAL_MS / 1000,
  });
  const intervalRef = useRef<NodeJS.Timeout | null>(null);
  const countdownRef = useRef<NodeJS.Timeout | null>(null);

  const fetchGraph = useCallback(async () => {
    setState((s) => ({ ...s, status: "loading" }));
    try {
      const response = await fetch(url, { cache: "no-cache" });
      if (!response.ok) throw new Error(`HTTP ${response.status}`);
      const data = await response.json();
      setState({
        data,
        status: "success",
        lastUpdated: new Date(),
        lastError: null,
        nextRefreshIn: REFRESH_INTERVAL_MS / 1000,
      });
    } catch (err) {
      setState((s) => ({
        ...s,
        status: "error",
        lastError: err instanceof Error ? err.message : String(err),
      }));
    }
  }, [url]);

  useEffect(() => {
    fetchGraph();
    intervalRef.current = setInterval(fetchGraph, REFRESH_INTERVAL_MS);
    countdownRef.current = setInterval(() => {
      setState((s) => ({ ...s, nextRefreshIn: Math.max(0, s.nextRefreshIn - 1) }));
    }, 1000);
    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current);
      if (countdownRef.current) clearInterval(countdownRef.current);
    };
  }, [fetchGraph]);

  const isStale =
    state.lastUpdated && Date.now() - state.lastUpdated.getTime() > STALE_THRESHOLD_MS;

  return { ...state, isStale, refresh: fetchGraph };
}

// Usage:
export function GraphViewer() {
  const { data, status, lastUpdated, nextRefreshIn, isStale, refresh } = useGraphWithAutoRefresh(
    "https://raw.githubusercontent.com/stsgs1980/Z-ai-platform/main/docs/_graph/id-graph.json",
  );
  if (status === "error" && !data) return <ErrorPanel onRetry={refresh} />;
  if (!data) return <LoadingPanel />;
  return (
    <div>
      <StatusBar
        status={status}
        lastUpdated={lastUpdated}
        isStale={isStale}
        nextRefreshIn={nextRefreshIn}
      />
      <GraphSVG nodes={data.ids} edges={data.related_edges} />
    </div>
  );
}
```

### Refresh UI: countdown to next refresh

```tsx
function StatusBar({ status, lastUpdated, isStale, nextRefreshIn }) {
  if (status === 'loading') return <Badge color="blue">Refreshing...</Badge>;
  if (status === 'error')   return <Badge color="red">Error — using cached data</Badge>;
  if (isStale)             return <Badge color="yellow">Stale (>15 min old)</Badge>;
  return (
    <div>
      <Badge color="green">Live</Badge>
      <span>Last updated: {lastUpdated?.toLocaleTimeString()}</span>
      <span>Next refresh: in {formatTime(nextRefreshIn)}</span>
    </div>
  );
}

function formatTime(seconds: number): string {
  if (seconds < 60) return `${seconds}s`;
  const m = Math.floor(seconds / 60);
  const s = seconds % 60;
  return `${m}m ${s}s`;
}
```

### Anti-patterns to avoid

```tsx
// BAD: refetch on every render (API spam)
useEffect(() => {
  fetchGraph();
});

// BAD: no error handling
const data = await fetch(url).then((r) => r.json());

// GOOD: proper error handling + keep cached data
try {
  const response = await fetch(url);
  if (!response.ok) throw new Error(`HTTP ${response.status}`);
  return await response.json();
} catch (err) {
  // Keep showing last known good data
  setState((s) => ({ ...s, error: err.message }));
}
```

```

T+0:30   GitHub Actions CI starts
T+1:00   verify-id-graph.js --json generates new data
```

## Stability contract

The JSON shape is governed by **verify-id-graph.js v1.1.x**. Within v1.1.x:

- `summary.*` fields are stable
- `ids[]`, `related_edges[]`, `aligned_with_edges[]` arrays are stable
- New fields MAY be added (consumers should ignore unknown fields)
- Field renames or removals require a major version bump (v2.0.0)

If you depend on this API, pin to a specific verifier version:

```
GET .../id-graph.json?v=1.1.6
```

When the script version changes, the `version` field in the response will
indicate the new shape. Consumers should check this field and handle migrations.

## Source of truth

This JSON is generated by `node standards/scripts/verify-id-graph.js --json`
in the `verify-id-graph.yml` GitHub Actions workflow, then committed back to
the repo on every push to main.

**The data in this file IS the state of Z-ai-platform at the corresponding
commit.** There is no transformation, no caching layer, no manual editing.

If `Z-ai-graph-viewer` shows X, that X is what is actually in Z-ai-platform.

## Pinning to a specific commit

For reproducibility, you can fetch from a specific commit SHA instead of `main`:

```
GET https://raw.githubusercontent.com/stsgs1980/Z-ai-platform/<commit-sha>/docs/_graph/id-graph.json
```

This is useful for:

- Archival views
- Reproducible builds
- Comparing versions
